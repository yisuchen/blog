Azure Logger

發表於2024-02-20|更新於2024-02-20|Line
|評論數:

前言

在練習串接 Line Message API 時會遇到, Line Webhook 只接受 https 的網址
如果偷懶, 可以利用 ngrok 打通網路環境
但在公司的網路環境中, 禁止使用 ngrok

問題

Webhook 接收到的格式長得如何!?
我想, 第一步也只能實際將 Linebot 架上之後, 才有辦法接收實際的 Line Webhook 訊息

獲得 Line Webhook 訊息格式後, 後續開發上就能方便的修改各項參數
再利用 postman 打到自己的 local 環境進行開發與偵錯

解法參考

我將 Linebot 上到 Azure 上

其他還有很多雲端空間可以參考, 不一定要綁死在 Azure 上

下面也記錄一下我取得 LineWebhook 的過程

  1. 安裝套件 Microsoft.Extensions.Logging.AzureAppServices

    dotnet add package Microsoft.Extensions.Logging.AzureAppServices

  2. 在 Program.cs 中注入服務

    builder.Logging.AddAzureWebAppDiagnostics();

  3. 在 appsettings.json 可以定義 logger 相關的設定, 這裡不詳述, 先知道有這項資訊就好
  4. 在 Controller 很簡單, 跟著 dotnet 原本的 logger 就好
  5. 簡易 Logger 紀錄範例
  6. 開啟 Azure , 進入 App Service 中的紀錄資料流, 即可獲得 Webhook 格式以及內容
