但是!!

Serilog 為了考慮效能與降低寫紀錄時對應用程式的影響
寫入 log 時, 會先存到一個緩衝區(Queue)
之後才會真正寫入 log 儲存器(自定, 可能 seq or txt…等等等)
所以 .NET 應用程式關閉前, 必須要確保 log 都有紀錄完成且儲存
必須要包 try/catch/finally

參考

Verbose = 0, (等同 .NET Trace) (最冗長)

Debug = 1, (同 .NET)

Information = 2, (同 .NET)

Warning = 3, (同 .NET)

Error = 4, (同 .NET)

Fatal = 5 (等同 .NET Critical ) (嚴重錯誤)
