Line Sticker 小心得

發表於2023-08-15|更新於2023-08-15|Line
|評論數:

前言

這篇應該沒甚麼內容, 簡單做個紀錄

Line Message Sticker

使用 Line Message Sticker 時, 有查到 Line 文件中有列出 Line Message API 提供的表情貼圖
Line Message API Sticker List

但是 Line 還有 2 個系列是最基礎的表情, 但似乎沒有呈現在列表上
但其實他們也可以使用在 Line Message API 中的

  1. Package ID : 1
  2. Package ID : 2

使用

其他系列在官方提供的文件中, 可以很快的看到表情的貼圖
但在 “系列1”、”系列2” 沒有那麼直覺的觀看
(不過可以直接打開手機中的　Line , 最基本的那2個貼圖就是系列1,2了)

另外, 如果你想要模仿官方製作一個聊天後台, 串接自己的產品
發送前, 至少要預覽準備發送的是甚麼貼圖
這時候, 大致有 2 個作法

  1. 把貼圖保存成自己的資源
  2. 官方提供的貼圖預覽

這時候有一個 API 其實在文件中我沒有看到, 但不知道從哪裡挖出來的, 剛好做個紀錄保存

  • https://stickershop.line-scdn.net/stickershop/v1/sticker/{stickerId}/{android/ios}/sticker.png
  • e.g.
    https://stickershop.line-scdn.net/stickershop/v1/sticker/2/android/sticker.png

這邊也只需要知道 StickerId
平台選擇上 , 建議不管平台, 一律使用 Android, ios 似乎有些圖片會破圖

後話

這邊單純只是記錄一些小心得, 產品的需求, 還是要以實際的使用狀況來決定
使用的人數 / 網路流量 / 快取的選用 / 本地的資源 …等

要將貼圖保存成自己的資源, 或是打 API 即時取得圖片資源, 或是其他做法
都是產品規劃上, 可以思考的方向
