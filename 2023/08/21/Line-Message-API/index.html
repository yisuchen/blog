前言 (Line UserId)

Line Message API , 顧名思義, 是用來與 Line 介接並操作一些動作的資源
在 Line 粉絲團裡, 每個用戶都會有獨立的 Id (UserId) e.g. U8189cf6745fc0d808977bdb0b9f22995
而要特別注意的一點, 這一個 UserId 並不是共享的
事實上
同一個使用者, 在不同的 Provider 中的 UserId, 皆為不同
以下看圖說明 (文件參考)

如果把 Provider 以公司來比喻
那 UserId 就是公司內, 員工的編號, 假設 A 員工同時在兩個不同公司內工作
淺而易見的, UserId(員工編號) 當然不一樣

另一個情況, X 集團內有 甲、乙、丙公司
今天 B 本來在甲公司上班, 因職務調動, 輾轉到乙、丙公司上班, 但因為同屬 X 集團, 所以他的 UserId 都維持一樣
(此處是對 Line UserId 舉例說明幫助快速瞭解, 實際情況可能有所不同, 請謹慎參考)

Line Webhook

前幾篇文章 Line Webhook Event 有一些 Webhook 的說明
這邊再做一些補充 :

webhook for 1-by-1 , 群組 或 多人聊天 :
1. Message : 接收訊息 (Text , Image , Video ….共 7 種 message 文件)
2. Unsend : 收回訊息
3. Follow : 加為好友 / 解除封鎖
4. Unfollow : 封鎖
5. Join : 被邀請至群組或聊天室
6. Leave : 從群組或聊天室被剃除
7. Member join : 用戶加入群組或聊天室
8. Member leave : 用戶離開群組或聊天室
9. Postback : 類似 Webhook , 使用者觸發某項條件, 並夾帶資料回傳回來的事件
10. Video viewing complete : 影片觀看完成

Line Message types

webhook 屬於被動接收的部分, 接下來讓我們來看看, 主動發送的部分
首先以下 9 種訊息類型 :
1. Text
2. Sticker
3. Image
4. Video
5. Audio
6. Location
7. ImageMap
8. Templete
9. Flex

以上這些, 照著官方文件, 填入對應的格式, 基本上沒甚麼問題
這邊記錄幾個使用心得

Audio Message

文件規格內有一項 duration , 這是用來顯示音訊時長

Location

文件規格內, Address 是用顯示的文字
latitude , longitude 是 GPS 位置

ImageMap Message (文件)

可以對圖片劃分區塊, 並設定不同的動作, 這邊記錄一下使用的小細節

  1. 配置圖片時要準備 240px、300px、460px、700px、1040px 這 5 種 png/jpeg 格式的圖片(缺一不可), 但不要有副檔名

  2. 檔名對應尺寸大小 , e.g : 700px 的圖片, 檔名為 ‘700’

  3. 放在同一個路徑內

  4. 圖片的 url 只需要指到資料夾路徑即可 e.g. : (baseUrl/folderName)

  5. 區塊的劃分要以基底的尺寸來做設定, 例如基底為 1040 x 1040
    area 劃分的起點位置 (x,y) 以及區塊大小 (width,height)
    皆由基底做為標準, 可參考以下範例

Templete Message (按鈕、確認、輪播、圖片輪播) (文件)

如果只需要一些簡單格式, 這一個訊息格式算是非常好用, 但使用的人越來越少…
如果需要樣式的自由性
推薦研究 Flex Message, 因為它能取代 Templete Message , 甚至做的更漂亮
此處先暫時不討論 Flex Message

訊息的共有屬性

接下來, 我想說說各訊息格式的共有屬性, 我比較少看到有人實作

  1. Sender
    可以更換傳送者的圖像,以及名稱, 假設今天粉絲團負責的客服人員是 A, 就可以切換成 A的圖像以及名稱 (可能讓客戶更有互動的感覺?)

    格式也非常簡單, 以 Text 為範例

  2. Quick Reply
    裡面最多可以埋 13 個 Quick Reply Buttons
    顧名思義, Buttons…感覺就像點擊了要做甚麼事, 所以可以聯想成 “動作”
    先來簡單看個格式範例 :

    意思是 items 內可以埋最多 13 個 Quick reply button
    item type 固定是 action
    其中可以改變的是 action type, 共有 7 種 actions 可以使用
    1. Postback action
    2. Message action
    3. URI action
    4. Datetime picker action
    5. Camera action
    6. Camera roll action
    7. Location action
    而不同的 action type 有不同的組合方式, 詳情參考 文件

後話

此處先暫時不討論 Flex Message , 主要是裡面的東西比較複雜, 一時之間不知道如何說明
或許會再下一次的文章中再補充說明

也會因為放在不同的地方, 使用的方式也不同, 或許還有一些限定條件
