Line Webhook Event

發表於2023-08-09|更新於2023-08-09|Line
|評論數:

前言

簡單紀錄關於 Line Message API Webhook 兩三事

Webhook 入口

在 Line 後臺設置 WebHook URL , 只能設定一個, 意味著
使用非同步處理 Webhook 事件, 是比較恰當的

Webhook 相關 API

  1. PUT https://api.line.me/v2/bot/channel/webhook/endpoint 更新 Webhook URL
  2. GET https://api.line.me/v2/bot/channel/webhook/endpoint 取得目前 Webhook 資訊
  3. POST https://api.line.me/v2/bot/channel/webhook/test 發送測試

其實以上…用到的機會很少, 基本上從 Line 開發者後台進行操作較為直覺

Webhook 驗證

Line 服務的 IP 可能是會變動的, 使用白名單不是一個良好的設定
Line 文件中也有提供開發者, 驗證 Webhook 是否由 Line 發送的
而驗證方法文件中也有提到
請參考

Webhook 的組成

Request body 內含

  1. destination(string) - 你的 LineBot 的 UserId
  2. events (Array) - Webhook 事件集合

特別注意! 同一個 Webhook 進來, 可能包含多個 events
通常使用 Linebot 常使用到的大約有 6 種
Webhook event 請參考

Webhook 使用

基本上 Webhook 的使用方式很簡單

  1. 接收 Line 傳遞來的 Webhook Request
  2. 驗證簽名
  3. 解析出 Webhook events 集合
  4. 針對個別類型的事件, 進行想要的操作

額外補充

Webhook message 事件中, 可能為 images , videos , audio , files
實際的資源並不會隨著 Webhook 一起傳遞進來
需要拿著 message : id , 呼叫另一隻 api 進行取得

GET https://api-data.line.me/v2/bot/message/{messageId}/content

呼叫任何 Line 時, Header 中必須包含

‘Authorization: Bearer {channel access token}’
